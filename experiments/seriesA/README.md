# Series A - does more power help? (compute scaled with `c₂`)

Impact variable: `c₂` (computational power).
Fix `M = 5` and scale gradient work with compute as `M·E = 25·c₂` (`E = 5·c₂`), for `c₂ ∈ {1, 2, 4, 8}`, plus an unconstrained `baseline`.
This holds the busy window fixed at `busy_steps = 1000` (50% drop), so the effect of `c₂` comes through the amount of learning, not the window size.
See Series C for the complementary fixed-work sweep that isolates the window.

## Run

```bash
uv run python -m ppo_study.run_experiment experiments/seriesA/config/cartpole.json \
    --n-runs 30 --n-proc 10 --root-dir experiments/seriesA/results/cartpole
uv run python -m ppo_study.run_experiment experiments/seriesA/config/acrobot.json \
    --n-runs 30 --n-proc 10 --root-dir experiments/seriesA/results/acrobot
```

Figures are generated by `make_figures.sh`. Outputs under `results/`/`figures/` are gitignored.
