# Series B - epochs vs. data? (same budget, vary the split)

Impact variable: allocation of a fixed budget `M·E = 25·c₂` between epochs and minibatches.
Vary `M ∈ {10, 5, 2}` for `c₂ ∈ {2, 4}`, plus an unconstrained `baseline`.
Series A's `M = 5` rows are Series B's middle column, so the two form one grid.
The busy window stays fixed at `busy_steps = 1000`.

## Run

```bash
uv run python -m ppo_study.run_experiment experiments/seriesB/config/cartpole.json \
    --n-runs 30 --n-proc 10 --root-dir experiments/seriesB/results/cartpole
uv run python -m ppo_study.run_experiment experiments/seriesB/config/acrobot.json \
    --n-runs 30 --n-proc 10 --root-dir experiments/seriesB/results/acrobot
```

Figures are generated by `make_figures.sh`. Outputs under `results/`/`figures/` are gitignored.
